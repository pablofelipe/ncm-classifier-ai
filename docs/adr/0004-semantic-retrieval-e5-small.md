# ADR-0004: Semantic Retrieval via ChromaDB + multilingual-e5-small

## Status
Accepted — 2026-06-11

## Context

ADR-0003 established the naive baseline (top-1 3.3%, top-3 16.7%).
This ADR isolates one variable: replacing naive retrieval with dense
semantic retrieval. Rerank remains Passthrough.

## Decision

- **Embedder**: `intfloat/multilingual-e5-small`, revision pinned
  `614241f622f53c4eeff9890bdc4f31cfecc418b3` (captured 2026-04-02,
  HF lastModified). Chosen over the MiniLM Chroma default (EN-only)
  and bge-m3 (hybrid dense+sparse — would add a second variable);
  selected for asymmetric retrieval training (`query:` / `passage:`
  prefixes) matching our short-query → descriptive-passage pattern.
- **E5EmbeddingFunction** (`src/retrieval/embedding.py`) enforces
  prefix separation by design: explicit `embed_documents` /
  `embed_query` methods, no symmetric `__call__`. Prefix application
  is validated by unit tests (spy encoder) and the asymmetry by an
  integration test (same text, different embeddings).
- **Composition root** (`src/api/dependencies.py`) rewired to
  `ChromaRetrievalAdapter`; an empty index raises an actionable
  `RuntimeError` ("run make index") — no silent fallback that would
  mask a misconfigured deployment.
- **NaiveRetrievalAdapter** retained in the repo as the ADR-0003
  baseline artifact (out of the active path).
- **CI**: two-step torch install (CPU-only index), Hugging Face cache
  keyed on a hash of `embedding.py` (model + revision pin → automatic
  cache bust), `make index` before the integration tier and eval gate.

## Baseline Measured (2026-06-11)

Eval set: `eval/v1_cases.json` (30 cases). Index: 34 Chapter 22 entries,
TIPI Decreto 11.158/2022 (consolidado em 08/06/2026).

| Metric | Naive (ADR-0003) | e5-small | Δ | v1 target |
|---|---|---|---|---|
| Top-1 | 1/30 = 3.3% | **10/30 = 33.3%** | +30.0 pp | ≥ 70% ✗ |
| Top-3 | 5/30 = 16.7% | **19/30 = 63.3%** | +46.6 pp | ≥ 90% ✗ |

Per-difficulty breakdown:

| Difficulty | Top-1 | Top-3 |
|---|---|---|
| easy (11) | 4/11 (36%) | 7/11 (64%) |
| medium (12) | 3/12 (25%) | 7/12 (58%) |
| hard (7) | 3/7 (43%) | 5/7 (71%) |

The hard > easy anomaly is not noise: hard cases carry long, technical
descriptions (whey supplement, oat beverage, 0.0 beer) that match the
rich indexed texts of 2202.99/2202.91, while several easy cases depend
on terms that do not occur in the indexed documents at all (see error
analysis).

Reachable-set accuracy: 10/29 = 34.5% top-1 (case-008 is
out-of-chapter by construction — the index covers Chapter 22 only;
structural ceiling is 29/30).

## Error Analysis (20 top-1 misses)

Full per-case data: `docs/adr/assets/0004-analysis.json` (generated by
`scripts/analyze_adr0004.py`, k=10 diagnostic depth).

| Category | Cases | Fraction |
|---|---|---|
| TIPI text poverty | 003, 014, 015, 016, 017, 018, 019, 020, 022, 023, 024, 026, 029 | **13/20 (65%)** |
| Negation blindness | 001, 005, 011 | 3/20 (15%) |
| Colloquial gap (pure) | 004, 025 | 2/20 (10%) |
| Frontier confusion (`confusion_chapters`) | 008, 009 | 2/20 (10%) |

**Key finding — TIPI text poverty (65%):** 8-digit TIPI descriptions
are relative to their parent heading; `build_document_text` indexes
them without hierarchical context. Literal evidence: 2204.21.00 reads
"Em recipientes de capacidade não superior a 2 l" — the word "vinho"
does not occur (only in the ex-tarifário "Vinhos da madeira, do porto
e de xerez"); "vermute" does not occur in any indexed document
(2205.10.00 is also "Em recipientes…"); "uísque" is absent from
2208.30.20 ("Em embalagens…"). Generic entries ("Outros"/"Outras")
carry no semantics. Side effect: 2208.30.10 acts as a universal
attractor — it is rich in "teor alcoólico / recipiente" terms present
in every e-commerce query, and shows up as a wrong top-1/top-2 in six
cases.

**Negation blindness (15%):** dense retrieval does not discriminate
negation — "sem gás, sem aromatizantes" matches "adicionadas de…
aromatizadas" (case-001); "Zero" and "teor alcoólico 5%" pull
"Cerveja sem álcool" (2202.91) to top-1 (cases 005, 011).

**Rerank ceiling:** 9 of the 20 misses have the correct NCM in
top-3 (ranking error), 11 miss top-3 entirely — 9 of those are not
even in the top-10. A perfect reranker over current retrieval caps at
19/30 = 63.3% top-1 — below the 70% target. Hierarchical document
enrichment must precede reranking.

## Known Limitation: colloquial vs fiscal language

First documented as xfail in
`tests/integration/retrieval/test_retrieve.py` (CP3,
`test_retrieves_colloquial_product_name`). Quantified here: 2/20
errors (10%) — real but not dominant, refuting the working hypothesis
that colloquial queries would dominate the error fraction. Cleanest
example: "cachaça" does not occur anywhere in TIPI Chapter 22 — the
entry reads "Rum e outras aguardentes provenientes da destilação,
após fermentação, de produtos da cana-de açúcar" (case-025, not in
top-10). The xfail stays in place (strict=False) until a future ADR
closes the gap.

## Confidence scores are uncalibrated

ECE = 0.54 (informational only; 1 − cosine distance is not a
probability). All 30 top-1 scores fall in [0.862, 0.899] — a single
histogram bin; mean score of hits (0.876) is indistinguishable from
misses (0.871), so the raw score does not separate right from wrong.
Threshold T cannot be calibrated on raw cosine — deferred to rerank
scores or a trained calibrator (future ADR). The current T=0.7 is a
placeholder with no semantics: every result labels "high" regardless
of correctness.

## Consequences

Positive:
- Top-1 accuracy improved 10× over the naive baseline with a single
  isolated variable; the eval-first discipline now has a meaningful
  reference point for the next interventions.
- CI is reproducible end-to-end: pinned model revision, automatic
  cache bust on pin change, index built per run, integration tier
  (including the documented XFAIL) green in ~2 min.
- Two adapters now validate `RetrievalPort` structurally (Naive and
  Chroma), confirming the port design.

Negative:
- v1 targets remain distant (top-1 33.3% vs ≥ 70%; top-3 63.3% vs
  ≥ 90%).
- Confidence scores carry no signal — the confidence gate is
  semantically inert until calibration lands.
- The out-of-chapter case (case-008) is structurally unreachable with
  a single-chapter index.

## Path Forward

- **ADR-0005 (recommended next): hierarchical document enrichment** —
  prepend parent heading/subheading descriptions in
  `build_document_text`; attacks the dominant error mode (65%) and
  the attractor effect; cheapest intervention.
- **ADR-0006 (candidate): LLM rerank** — only meaningful after
  enrichment lifts the retrieval ceiling above the target.
- Later: hybrid dense+sparse (bge-m3), query expansion, multi-chapter
  index (unlocks case-008).
